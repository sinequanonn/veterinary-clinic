# 환자 정보 관리 시스템

---
## 기능 사항 정의
### 도메인
- #### Vet 엔티티 구현
  - [x] 수의사명 유효성 검증
- #### Pet 엔티티 구현
  - [x] 환자명 유효성 검증
  - [x] 보호자명 유효성 검증
  - [x] 품종 유효성 검증
  - [x] 체중 유효성 검증
  - [x] 생년월일 유효성 검증
- #### MedicalRecord 엔티티 구현
  - [x] 진료 날짜 유효성 검증
  - [x] 주관적 판단 유효성 검증
  - [x] 객관적 판단 유효성 검증
  - [x] 확정 진단명 유효성 검증
  - [x] 진료 계획 유효성 검증
- #### MedicalReport 엔티티 구현
  - [x] 진료 소견서 작성일 유효성 검증
  - [x] 주요 증상 유효성 검증
  - [x] 확정 진단명 유효성 검증
  - [x] 진료 계획 유효성 검증
  - [x] 추후 관리 유효성 검증

### 서비스
#### 수의사 관리
- [x] 수의사는 시스템에 회원으로 등록한다.
- [ ] 수의사 목록을 확인한다.

#### 환자(동물) 관리
- [ ] 시스템에 환자를 등록한다.
- [ ] 환자 목록을 조회한다.
- [ ] 하나의 환자 상세 정보를 조회한다.
- [ ] 환자 정보를 수정한다.
- [ ] 환자 정보를 삭제한다.

#### 진료 기록 관리
- [ ] 수의사는 진료를 본 환자에 대한 진료 기록을 작성한다.
- [ ] 한 환자의 모든 진료 기록 목록을 조회한다.
- [ ] 하나의 진료 기록의 상세 정보를 조회한다.
- [ ] 진료 기록을 수정한다.
- [ ] 진료 기록을 삭제한다.

#### 진료 소견서 관리
- [ ] 수의사는 한 환자에 대한 진료 소견서를 작성한다.
- [ ] 하나의 환자의 진료 소견서 목록을 조회한다.
- [ ] 하나의 진료 소견서 상세 내용을 조회한다.
- [ ] 진료 소견서를 수정한다.
- [ ] 진료 소견서를 삭제한다.

## 용어 사전
| 한글명    | 영문명                | 설명                             |
|--------|--------------------|--------------------------------|
|수의사|vet| 동물병원에서 진료를 하는 의사               |
| 환자     | pet                | 동물병원에서 진료를 받는 동물               |
| 보호자    | owner              | 환자를 키우는 반려인                    |
| 환자 진료  | medical record     | 환자의 진료 내역을 SOAP 형식으로 기록한 문서    |
| 주관적 판단 | subjective         | 반려인이 관찰한 증상(SOAP의 S)           
| 객관적 판단 | objective          | 수의사의 신체검사와 측정 결과(SOAP의 O)      
| 확정 진단명 | assessment         | 수의사가 내린 최종 진단명(SOAP의 A)        
| 진료 계획  | plan       | 진료 치료 방법 및 처방 내용(SOAP의 P)      
| 진료 소견서 | medical report     | 수의사의 소견이 들어간 환자의 진료 결과를 정리한 문서 
| 환자 정보  | signalment         | 환자의 기본 정보(이름, 품종, 나이, 체중, 보호자) 
| 주요 증상  | chief complaint    | 반려인이 관찰한 주된 증상과 진료 방문 이유       
| 추후 관리  | postoperative care | 퇴원 후 보호자가 수행해야 할 관리 사항
